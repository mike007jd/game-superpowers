## TikTok short structure

### Hook

“Same model. Same task. One prompt difference.”

### Shot order

1. show the baseline prompt screenshot
2. show the broken / weaker output
3. cut to the Game Superpowers prompt screenshot
4. show the playable result
5. show the comparison table quickly
6. end on the repo URL

### Voiceover draft

“I built an open-source game workflow system called Game Superpowers.
This is the same FPS task on the same model.
On the left is the raw run.
On the right is the run using `/using-game-superpowers`.
The point is not that every run will look exactly like this.
The point is that workflow quality changes first-prompt outcomes.
The repo includes the screenshots, the HTML files, and a benchmark case study.”

## Visual tips

- Lead with the side-by-side result, not the repo screenshot.
- Put the model name on screen: `MiniMax-M2.7-highspeed`.
- Put “same task” and “first prompt” on screen early.
- Keep the repo URL visible in the closing shot.
- Invite people to run their own benchmark and post results.
